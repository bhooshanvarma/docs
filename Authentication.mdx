description: "---\ntitle: \"Authentication\"\ndescription: \"How Layer3x authenticates your agents\"\n---\n\n## API Key Authentication\n\nEvery agent registered in Layer3x has a unique API key.\nInclude it in every request as a header:\n```\nX-Agent-API-Key: agk_your_key_here\n```\n\nAPI keys are:\n\n- Generated once at agent creation\n- Shown only once — store them immediately\n- Rotatable at any time from the Agents page\n- Scoped to a single agent and tenant — keys cannot access other agents\n\n## HMAC Request Signing (optional)\n\nFor higher security, Layer3x supports HMAC-SHA256 request signing per agent — the same pattern used by Stripe and Plaid.\n\nWhen enabled, include this header on every request:\n```\nX-Layer3-Signature: sha256=<hex_signature>\n```\n\nCompute the signature in your code:\n```javascript\nconst signature = crypto\n  .createHmac('sha256', YOUR_HMAC_SECRET)\n  .update(JSON.stringify(requestBody))\n  .digest('hex')\n```\n\n> **Note:** HMAC signing is opt-in per agent. Agents without an HMAC secret configured authenticate with their API key only.\n\n## Key Rotation\n\n1. Go to **Agents** → click **···** on the agent row\n2. Click **Rotate API Key**\n3. Copy the new key immediately — the old key is invalidated\n\nAfter rotation, update any systems using the old key before it expires."
