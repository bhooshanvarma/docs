description: "---\ntitle: \"Authentication\"\ndescription: \"How Layer3x authenticates your agents\"\n---\n\n## API Key Authentication\n\nEvery agent registered in Layer3x has a unique API key. \nInclude it in every request as a header:\n```bash\nX-Agent-API-Key: agk_your_key_here\n```\n\nAPI keys are:\n- Generated once at agent creation\n- Shown only once — store them immediately\n- Rotatable at any time from the Agents page\n- Scoped to a single agent and tenant — keys cannot access other agents\n\n## HMAC Request Signing (optional)\n\nFor higher security requirements, Layer3x supports \nHMAC-SHA256 request signing per agent — the same pattern \nused by Stripe and Plaid for webhook verification.\n\nWhen HMAC signing is enabled for an agent, every request \nmust include a signature header:\n```bash\nX-Layer3-Signature: sha256=<hex_signature>\n```\n\nThe signature is computed as:\n```javascript\nconst signature = crypto\n  .createHmac('sha256', YOUR_HMAC_SECRET)\n  .update(JSON.stringify(requestBody))\n  .digest('hex')\n```\n\n<Note>\n  HMAC signing is opt-in per agent. Agents without an HMAC \n  secret configured authenticate with their API key only.\n</Note>\n\n## Key Rotation\n\nRotate your agent's API key from the Agents page:\n\n1. Go to **Agents** → click **···** on the agent row\n2. Click **Rotate API Key**\n3. Copy the new key immediately — the old key is invalidated\n\nAfter rotation, update any systems using the old key before \nthe rotation is complete."
